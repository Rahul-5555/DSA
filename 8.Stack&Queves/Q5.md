# üß± Min Stack (LeetCode 155)

## üìå Problem Statement

Design a stack that supports:

- `push(val)`
- `pop()`
- `top()`
- `getMin()`

All operations must run in **O(1)** time complexity.

---

## üí° Approach

Instead of storing only the value in the stack,  
we store a pair:

```
[value, minimum_till_now]
```

This allows us to retrieve the minimum element in constant time.

---

## üß† Logic Explanation

When pushing a new value:

- If stack is empty ‚Üí push `[val, val]`
- Else:
  ```
  minVal = Math.min(val, lastMin)
  push [val, minVal]
  ```

This ensures every element knows the minimum value up to that point.

---

## üßæ Implementation (JavaScript)

```javascript
var MinStack = function () {
  this.s = [];
};

MinStack.prototype.push = function (val) {
  if (this.s.length === 0) {
    this.s.push([val, val]);
  } else {
    let minVal = Math.min(val, this.s[this.s.length - 1][1]);
    this.s.push([val, minVal]);
  }
};

MinStack.prototype.pop = function () {
  this.s.pop();
};

MinStack.prototype.top = function () {
  return this.s[this.s.length - 1][0];
};

MinStack.prototype.getMin = function () {
  return this.s[this.s.length - 1][1];
};
```

---

## üîç Example

```javascript
var minStack = new MinStack();

minStack.push(-2);
minStack.push(0);
minStack.push(-3);

console.log(minStack.getMin()); // -3
minStack.pop();
console.log(minStack.top());    // 0
console.log(minStack.getMin()); // -2
```
